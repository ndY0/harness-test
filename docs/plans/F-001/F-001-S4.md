## Context

This sub-task implements three distinct but leaf-level concerns that have no
mutual dependencies and all depend only on the value types (F-001-S0):

- **physics.rs** — movement resolution with tunnel wrapping; collision detection
- **scoring.rs** — score accumulator, ghost-chain counter, extra-life flag
- **ui/** — Ratatui rendering (maze widget, HUD widget, size-check widget)

All three are consumed by F-001-S5 (the integration/wiring pass that writes
app.rs and main.rs). They must compile against the value types from S0 and may
forward-declare the `Maze`/`Player`/`Ghost`/`PelletMap`/`ScoreState`/`GameTimers`
structs as needed (using `use crate::game::...`).

## Inputs from spec

### physics.rs

From `docs/architecture/game-domain.md` §2.6 (Tunnel Wrapping) and §4:

```rust
// src/game/physics.rs
// apply_move: attempt to move entity at pos in direction dir within maze.
// Returns Some(new_pos) on success, None if the move is blocked by a wall.
// Tunnel wrapping is applied when pos is at col 0/20 and maze.tunnel_row(row).
pub fn apply_move(pos: Position, dir: Direction, maze: &Maze) -> Option<Position>;

// check_player_ghost_collision: returns true if player_pos matches any ghost pos.
// Ghost house ghosts (is_in_house) are excluded from collision.
pub fn check_player_ghost_collision(
    player_pos: Position,
    ghosts: &[Ghost],
) -> Option<usize>;  // index of colliding ghost, or None
```

Tunnel wrap logic (from architecture §2.6):
```rust
let new_col = (pos.col as isize + delta_col)
    .rem_euclid(maze.cols() as isize) as usize;
```
Applied only when `pos.col == 0 && dir == Left` or `pos.col == 20 && dir == Right`
AND `maze.tunnel_row(pos.row)` is true. If the wrap destination is a wall,
return `None` (movement blocked).

**Simultaneous collision edge case** (spec §Edge Cases):
If the player shares a tile with two ghosts, one frightened and one not:
the non-frightened ghost takes priority — player loses a life, frightened ghost
is not consumed. The function returns the index of the non-frightened ghost first.

### scoring.rs

From `docs/architecture/game-domain.md` §2.4 and §2.10:

| Event | Points |
|-------|--------|
| Eat dot | 10 |
| Eat power pellet | 50 |
| Eat 1st frightened ghost | 200 |
| Eat 2nd frightened ghost | 400 |
| Eat 3rd frightened ghost | 800 |
| Eat 4th frightened ghost | 1 600 |

Ghost-chain counter (`ghost_chain: u8`) resets on:
- Frightened mode expiry
- Player death
- New power pellet eaten (chain resets to 0)

Score is stored as `u32` (never `i32`). Extra life awarded at 10 000 points,
at most once per game (`extra_life_awarded: bool`).

```rust
// src/game/scoring.rs
pub struct ScoreState {
    pub score: u32,
    pub ghost_chain: u8,
    pub extra_life_awarded: bool,
}

impl ScoreState {
    pub fn new() -> Self;
    pub fn add_dot(&mut self);            // +10
    pub fn add_power_pellet(&mut self);   // +50, resets chain to 0
    pub fn eat_ghost(&mut self) -> u32;   // 200 * 2^chain, increments chain
    pub fn reset_chain(&mut self);        // called on death or frightened expiry
    /// Returns true if extra life should be awarded (score >= 10000, not yet awarded).
    pub fn check_extra_life(&mut self) -> bool;
}
```

### ui/mod.rs

From `docs/architecture/game-domain.md` §2.8:

```rust
pub const MIN_COLS: u16 = 80;
pub const MIN_ROWS: u16 = 30;
```

```rust
// ui/mod.rs
// Top-level render function. Takes a Ratatui Frame and the current App state.
// If terminal is too small, renders the error widget. Otherwise renders HUD + maze.
pub fn render(frame: &mut ratatui::Frame, app: &crate::app::App);
```

The `render` function and all sub-functions it calls take only `&` (immutable)
references. No mutable access inside the render path.

Size check: `frame.size().width < MIN_COLS || frame.size().height < MIN_ROWS`
→ render error widget:
```
Terminal too small.
Need at least 80×30. Current: <cols>×<rows>.
Please resize and the game will resume automatically.
```

### ui/renderer.rs

From `docs/architecture/game-domain.md` §7 (Ghost Rendering) and §9 (Layout):

```rust
// Returns a Ratatui widget that renders the 21×21 maze grid.
// Player cell: 'C' yellow foreground.
// Ghost cells: 'ᗣ' (U+15E3) with per-personality foreground:
//   Blinky=Red, Pinky=Magenta, Inky=Cyan, Clyde=Yellow
// Ghost frightened: 'ᗣ' Blue foreground.
// Ghost frightened blinking (last 30 ticks): alternate Blue/White each tick.
// ASCII fallbacks: B/P/I/C for ghosts, C for player.
// Dot: '.' White. PowerPellet: 'o' White bold. Wall: '#' (no colour). Empty: ' '.
pub fn maze_widget<'a>(game_state: &'a crate::app::GameState) -> impl ratatui::widgets::Widget + 'a;
```

The maze is centered in the available area using Ratatui's layout primitives.

## Acceptance criteria (subset)

- `apply_move` returns None for wall targets and wraps correctly in tunnel rows
  of stages 3 and 7
- `check_player_ghost_collision` returns the non-frightened ghost index when one
  frightened and one non-frightened ghost share the player's tile
- `ScoreState::eat_ghost()` returns 200, 400, 800, 1600 for chain counts 0–3
- `ScoreState::check_extra_life()` returns true once (and only once) when score
  first crosses 10 000
- `render()` shows the error widget when terminal < 80×30, normal game otherwise
- HUD score is zero-padded to 5 digits; stage is 1-indexed
- Ghost frightened rendering uses blue foreground; blinking alternates blue/white
