## Context

This sub-task implements two closely related game entities:

- `Player` ‚Äî the struct that tracks player position, facing direction, buffered
  input, and remaining lives. It is the direct target of keyboard input and the
  entity that physics moves each tick.
- `PelletMap` ‚Äî the mutable eaten-state overlay for dots and power pellets. The
  `Maze` struct is immutable after parse; `PelletMap` tracks which pellets are
  still active as a flat `Vec<bool>` indexed by `row * 21 + col`.

Both types are consumed directly by F-001-S4 (physics, scoring, UI), and by
F-001-S5 (app tick loop). They must compile correctly against the value types
from F-001-S0.

  set by `buffer_direction` on keypress and consumed by `consume_direction` on

- **Criterion 5**: When a player tile contains a dot, `pellets.eat(pos)` marks
  it eaten and returns `Cell::Dot`. When all pellets are eaten,
  `pellets.all_eaten()` returns true.

- **Criterion 6**: When `pellets.eat(pos)` returns `Cell::PowerPellet`,
  frightened mode is triggered (in the app loop ‚Äî not in this sub-task).

- **Criterion 9**: Player starts with 3 lives. The lives field is `u8`.
  `Player::lose_life()` decrements it. `Player::is_dead()` returns true when
  lives == 0.

- **Criterion 2.7**: Lives rendered as `‚ô•` symbols in the HUD (not in this
  sub-task).

### PelletMap initialisation

`PelletMap::new(maze: &Maze) -> PelletMap` iterates the maze's cell grid and
sets `true` for each `Cell::Dot` and `Cell::PowerPellet` tile. Indexed by
`row * 21 + col`. A `false` entry means that pellet has been eaten. A cell that
was never a pellet is also `false` in the map (it is simply never eaten).

`active_positions()` returns an iterator over `Position` values where the entry
is `true`.

### Player start position

`Player::new(start: Position) -> Player` creates a player at the given position
with 3 lives, no facing direction, and no buffered direction. The `start`
position comes from `Maze::player_start()`.

## Acceptance criteria (subset)

- `Player::buffer_direction(dir: Direction)` sets `buffered_dir` to `Some(dir)`
- `Player::consume_direction() -> Option<Direction>` takes and clears
  `buffered_dir`, returning the value
- `Player::lose_life()` decrements `lives` (saturating at 0 ‚Äî never wraps)
- `Player::is_dead() -> bool` returns `true` when `lives == 0`
- `PelletMap::new(&Maze)` correctly initialises pellet presence from maze data
- `PelletMap::eat(pos) -> Cell` marks that position eaten (if it was a pellet)
  and returns the cell type that was there; returns `Cell::Empty` if pos was not
  a pellet
- `PelletMap::all_eaten() -> bool` returns true when no active pellets remain
- `PelletMap::active_positions()` iterates all uneaten pellet positions

## Interface contracts

```rust
// src/game/player.rs
use crate::game::{Direction, Position};

pub struct Player {
    pub pos: Position,
    pub facing: Option<Direction>,
    pub buffered_dir: Option<Direction>,
    pub lives: u8,
}

impl Player {
    pub fn new(start: Position) -> Self;
    pub fn buffer_direction(&mut self, dir: Direction);
    pub fn consume_direction(&mut self) -> Option<Direction>;
    pub fn lose_life(&mut self);
    pub fn is_dead(&self) -> bool;
}
```

```rust
// src/game/pellets.rs
use crate::game::{Cell, Position};
use crate::game::maze::Maze;

pub struct PelletMap {
    data: Vec<bool>,
    cols: usize,
}

impl PelletMap {
    pub fn new(maze: &Maze) -> Self;
    pub fn eat(&mut self, pos: Position) -> Cell;
    pub fn all_eaten(&self) -> bool;
    pub fn active_positions(&self) -> impl Iterator<Item = Position> + '_;
}
```
