## Inputs from spec

### main.rs — TerminalGuard and event loop

From `docs/architecture/game-domain.md` §5 (Terminal Guard):

```rust
struct TerminalGuard;

impl TerminalGuard {
    fn new() -> anyhow::Result<Self> {
        crossterm::terminal::enable_raw_mode()?;
        crossterm::execute!(
            std::io::stdout(),
            crossterm::terminal::EnterAlternateScreen
        )?;
        Ok(Self)
    }
}

impl Drop for TerminalGuard {
    fn drop(&mut self) {
        let _ = crossterm::terminal::disable_raw_mode();
        let _ = crossterm::execute!(
            std::io::stdout(),
            crossterm::terminal::LeaveAlternateScreen
        );
    }
}
```

The event loop (100 ms tick, spec §Technical Notes):

```rust
fn main() -> anyhow::Result<()> {
    let _guard = TerminalGuard::new()?;
    let backend = CrosstermBackend::new(std::io::stdout());
    let mut terminal = Terminal::new(backend)?;
    let mut app = App::new();

    loop {
        terminal.draw(|frame| ui::render(frame, &app))?;
        if crossterm::event::poll(std::time::Duration::from_millis(100))? {
            match crossterm::event::read()? {
                Event::Key(key) => {
                    if let Some(action) = input::map_key(key) {
                        if app.handle_input(action) { break; }
                    }
                }
                Event::Resize(_, _) => { /* render loop handles it */ }
                _ => {}
            }
        }
        app.tick();
        if app.should_quit() { break; }
    }
    Ok(())
}
```

### app.rs — App state machine

```rust
pub enum AppState {
    Playing,
    Paused,
    GameOver,
    Victory,
}

pub struct App {
    pub state: AppState,
    pub game: GameState,
    tick_count: u64,
}

pub struct GameState {
    pub stage_index: usize,     // 0-indexed internally; HUD shows +1
    pub maze: Maze,
    pub pellets: PelletMap,
    pub player: Player,
    pub ghosts: [Ghost; 4],
    pub score: ScoreState,
    pub timers: GameTimers,
    pub dots_eaten: u32,        // for ghost house exit thresholds
    pub status_msg: String,     // "PAUSED" | "READY!" | "+ EXTRA LIFE!" | ""
}
```

Full `App::tick()` logic (one 100 ms tick):

1. Increment `tick_count`.
2. If `state != Playing`, return early (no game logic in Paused/GameOver/Victory).
3. If `timers.is_in_death_pause()`, skip physics and ghost AI; decrement timer
   via `timers.tick()` and handle `DeathPauseExpired`.
4. Consume player direction buffer → attempt `physics::apply_move`.
5. Check pellet at new player position: call `pellets.eat(pos)`:
   - `Cell::Dot` → `score.add_dot()`, increment `dots_eaten`
   - `Cell::PowerPellet` → `score.add_power_pellet()`, `timers.start_frightened()`,
     set all ghosts to `GhostMode::Frightened`, reset chain
6. Step each ghost (calls `ghost.step(...)` with tick_count).
7. Check player-ghost collision (`physics::check_player_ghost_collision`):
   - Non-frightened ghost → `player.lose_life()`, if dead → transition to
     GameOver; else `timers.start_death_pause()`, set status_msg = "READY!",
     reset all positions to stage start, reset chain.
   - Frightened ghost → `score.eat_ghost()`, `ghost.send_home()`.
8. Check `pellets.all_eaten()` → if true, advance to next stage (or Victory if
   stage_index == 9).
9. Call `timers.tick()` → handle events:
   - `FrightenedExpired` → set all ghosts back to previous mode, `score.reset_chain()`
   - `FrightenedBlinking` → note for renderer (tick_count parity drives blink)
   - `DeathPauseExpired` → clear status_msg
   - `ExtraLifeDisplayExpired` → clear status_msg
   - `PhaseChanged(mode)` → set all non-frightened ghosts to new mode
10. Call `score.check_extra_life()` → if true, `player.lives += 1`,
    `timers.start_extra_life_display()`, status_msg = "+ EXTRA LIFE!".

`App::handle_input(action: GameAction) -> bool` (returns true if should quit):
- `GameAction::Quit` → return true (quit)
- `GameAction::Pause` + Playing → transition to Paused, status_msg = "PAUSED"
- `GameAction::Pause` + Paused → transition to Playing, status_msg = ""
- `GameAction::MoveUp/Down/Left/Right` + Playing → buffer direction in player
- In GameOver or Victory state: any key except Pause → return true (quit)

Stage transition:
```rust
fn advance_stage(&mut self) {
    self.game.stage_index += 1;
    if self.game.stage_index >= 10 {
        self.state = AppState::Victory;
        return;
    }
    // reload maze, reset pellets, reset player/ghost positions, reset timers
    // preserve score and lives
    let rows = stages::STAGES[self.game.stage_index];
    self.game.maze = Maze::parse(rows);
    self.game.pellets = PelletMap::new(&self.game.maze);
    let start = self.game.maze.player_start();
    self.game.player.pos = start;
    self.game.player.buffered_dir = None;
    self.game.player.facing = None;
    self.game.ghosts = Self::make_ghosts(&self.game.maze);
    self.game.timers = GameTimers::new();
    self.game.dots_eaten = 0;
}
```

### tests/integration_test.rs

Write unit/integration tests exercising (without a live terminal):

1. `cargo build --release` success is verified by the build step itself.
2. `stages::STAGES` has exactly 10 entries, each exactly 21 rows of 21 chars.
3. `map_key` returns correct `GameAction` for all 14 mapped keys.
4. `Player::buffer_direction` / `consume_direction` round-trip.
5. `PelletMap::eat` marks eaten and `all_eaten` fires correctly.
6. Power pellet starts frightened, `ScoreState::eat_ghost` produces correct
   chain values (200, 400, 800, 1600).
7. Ghost collision with non-frightened ghost → life loss.
8. `Player::lose_life` at 0 lives → `is_dead()` true.
9. Extra life awarded once at 10 000 points.
10. `GameTimers::tick()` emits `FrightenedExpired` after 100 ticks.
11. `apply_move` blocked by wall, wraps correctly in tunnel row.
12. Tunnel wrapping only in stages 3 and 7 (check `maze.tunnel_row(10)` for
    each of the 10 stage mazes).
13. `ScoreState::check_extra_life` returns true once and false on repeat calls.
14. `GameTimers` scatter→chase→scatter→chase cycle emits correct `PhaseChanged`
    events after 70 and 200 ticks respectively; after two full cycles, no more
    `PhaseChanged` with Scatter.
15. `Ghost::send_home` resets position to home_pos and clears frightened mode.
16. HUD widget renders score as zero-padded 5-digit string.
17. `render` (mocked frame) shows error widget when cols < 80.
18. `App::handle_input(GameAction::Quit)` returns true.

Tests must all pass with `cargo test`.

## Acceptance criteria (subset)

All 18 parent spec acceptance criteria must be satisfied in the integrated binary.
Specifically:

- AC 1: `cargo build --release` succeeds.
- AC 2: 10 stages, each unique layout.
- AC 3–4: Arrow and WASD keys buffer directions; movement respects walls.
- AC 5: Dots eaten, stage advances when all eaten.
- AC 6: Power pellet → 100-tick frightened; last 30 blink.
- AC 7: Frightened ghost eaten → chain scoring; ghost sent home.
- AC 8: Non-frightened ghost collision → life lost; 10-tick freeze with "READY!".
- AC 9: 3 starting lives; extra life at 10 000 (once); `♥` in HUD.
- AC 10: GameOver screen on 0 lives; any key exits.
- AC 11: Victory screen on stage 10 clear; any key exits.
- AC 12: Score zero-padded 5 digits in HUD.
- AC 13: Stage number 1-indexed in HUD.
- AC 14: Esc or Q exits from any state; terminal restored.
- AC 15: P or Space toggles pause; physics skipped while paused.
- AC 16: Error widget when terminal < 80×30; resumes automatically.
- AC 17: No audio / windowing dependencies.
- AC 18: Terminal restored on all exit paths (TerminalGuard Drop).
