## Context

This is the foundation sub-task. It creates the Rust crate from scratch so that
every subsequent sub-task can branch from a compilable (or at least parse-able)
starting point. It owns exactly three files:

- `Cargo.toml` — the package manifest with all permitted dependencies
- `src/game/mod.rs` — the value types and re-exports shared by every other game
  module; no logic lives here
- `src/input.rs` — stateless key-event → GameAction mapping

No other sub-task may touch these files. Later sub-tasks import the value types
defined here without modification.

## Inputs from spec

From `docs/specs/pacman-terminal-game.md` §Technical Notes:

```toml
[package]
name = "pacman-terminal"
version = "0.1.0"
edition = "2021"

[dependencies]
ratatui   = "0.27"
crossterm = "0.27"
anyhow    = "1"
```

No other runtime dependencies are permitted.

From `docs/architecture/game-domain.md` §4 (Module Structure):

```
src/game/mod.rs  — Re-exports. Value types: Direction, Cell, GhostMode,
                   GameAction, Position. No logic.
src/input.rs     — map_key(KeyEvent) -> Option<GameAction>. Pure mapping,
                   no state.
```

### Value types to define in src/game/mod.rs

```rust
// Position: row/col integer coordinates
pub struct Position { pub row: usize, pub col: usize }

// Direction: the four cardinal directions
pub enum Direction { Up, Down, Left, Right }

// Cell: what a maze tile contains
pub enum Cell { Wall, Dot, PowerPellet, PlayerStart, GhostHouse, Empty }

// GhostMode: the three behavioural modes a ghost can be in
pub enum GhostMode { Chase, Scatter, Frightened }

// GameAction: semantic game commands produced by input mapping
pub enum GameAction {
    MoveUp, MoveDown, MoveLeft, MoveRight,
    Pause, Quit,
}
```

These types must derive `Debug`, `Clone`, `Copy`, `PartialEq` where applicable.

### Key mapping in src/input.rs

```
Arrow Up    → GameAction::MoveUp
Arrow Down  → GameAction::MoveDown
Arrow Left  → GameAction::MoveLeft
Arrow Right → GameAction::MoveRight
W / w       → GameAction::MoveUp
A / a       → GameAction::MoveLeft
S / s       → GameAction::MoveDown
D / d       → GameAction::MoveRight
P / p / Space → GameAction::Pause
Q / q / Esc   → GameAction::Quit
```

The function signature is:
```rust
pub fn map_key(event: crossterm::event::KeyEvent) -> Option<GameAction>
```

## Acceptance criteria (subset)

  must compile error-free)

- `src/input.rs` exports `map_key` and covers all key bindings from criteria 3,
  4, 14, and 15 of the parent spec
