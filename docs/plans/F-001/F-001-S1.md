## Context

This sub-task implements the static maze representation and embeds all 10 stage
layouts as compile-time string constants. It is the data layer that every other
game module reads — physics, ghosts, player, pellets, and the renderer all need
`Maze`. The layouts are verbatim from `docs/architecture/game-domain.md` §8 and
must not be altered.

## Inputs from spec

From `docs/architecture/game-domain.md` §2.2 (Stage Layout Format):

- Each stage is `const &[&str; 21]` of 21-character rows, 21 rows total.
- Cell encoding:
  - `#` → Wall
  - `.` → Dot
  - `o` → PowerPellet
  - `P` → PlayerStart
  - `G` → GhostHouse
  - ` ` → Empty path

The `STAGES` constant in `stages.rs` is `[&[&str; 21]; 10]`, indexed 0–9.

From `docs/architecture/game-domain.md` §4 (Module Structure):

```
src/game/maze.rs   — Maze struct: Vec<Vec<Cell>>. Methods: is_wall(pos),
                     is_ghost_house(pos), tunnel_row(row) -> bool.
                     Stage ASCII → Maze parsing lives here.
src/game/stages.rs — STAGES: [&[&str; 21]; 10] const array. One entry per
                     stage. No logic — pure data.
```

## Acceptance criteria (subset)

- `stages::STAGES` is `[&[&str; 21]; 10]` — exactly 10 entries, each exactly
  21 rows, each row exactly 21 characters (tunnel rows in stages 3 and 7 are
  the only exception: they use space chars at col 0 and col 20 rather than `#`)
- `Maze::parse(stage_rows: &[&str; 21]) -> Maze` successfully converts ASCII
  to a `Vec<Vec<Cell>>` grid using the cell encoding table
- `Maze::is_wall(pos) -> bool` returns true for cells containing `Cell::Wall`
- `Maze::is_ghost_house(pos) -> bool` returns true for `Cell::GhostHouse` tiles
- `Maze::tunnel_row(row: usize) -> bool` returns true for the tunnel rows in
  stages 3 (row 10) and 7 (row 10) — method checks if col 0 and col 20 of
  that row are both non-wall after parse
- `Maze::player_start() -> Position` returns the position of the `P` cell
- `Maze::ghost_house_center() -> Position` returns the center of the `GG` block

## Interface contracts

```rust
// src/game/maze.rs
pub struct Maze {
    pub cells: Vec<Vec<crate::game::Cell>>,
}

impl Maze {
    pub fn parse(rows: &[&str; 21]) -> Self;
    pub fn cols(&self) -> usize;   // always 21
    pub fn rows_count(&self) -> usize; // always 21
    pub fn is_wall(&self, pos: crate::game::Position) -> bool;
    pub fn is_ghost_house(&self, pos: crate::game::Position) -> bool;
    pub fn tunnel_row(&self, row: usize) -> bool;
    pub fn player_start(&self) -> crate::game::Position;
    pub fn ghost_house_center(&self) -> crate::game::Position;
}

// src/game/stages.rs
pub const STAGES: [&[&str; 21]; 10];
```
