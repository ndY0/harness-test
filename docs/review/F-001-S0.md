### NON_BLOCKING findings

- ID: NB1
- Type: style
- Observation: `map_key` silently drops events that carry `CONTROL` or `ALT` modifiers (lines 13–15 of `src/input.rs`). The spec does not require this guard. It is a sensible defence, but it is undocumented behaviour relative to the spec and could become surprising if a later sub-task needs to handle, e.g., Ctrl+C. The comment notes that plain Shift is handled implicitly by the `Char` variants, which is correct.
- Suggestion: Add a brief doc-comment on `map_key` that records this modifier-filtering contract so later Implementers are aware of it.

- ID: NB2
- Type: SPEC_GAP
- Observation: The sub-task spec says "derive `Debug`, `Clone`, `Copy`, `PartialEq` where applicable." `Eq` and `Hash` are not required by the spec but are frequently useful on enum types (e.g. `GameAction`, `Cell`) when these types appear as `HashMap` keys or in `HashSet`s in later sub-tasks.
- Suggestion: The Domain Architect may wish to add `Eq` (at minimum) to the derivation list in the spec for those types where `PartialEq` is already derived — they are trivially equivalent for plain enums.
